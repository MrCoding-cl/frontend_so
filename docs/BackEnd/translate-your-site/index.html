FibberRoutes

En esta area del codigo se ordenan las peticiones que pueden hacerse al programa.

Imports#

FiberRoutes.go
import (
"github.com/gofiber/fiber/v2"
"github.com/gofiber/fiber/v2/middleware/cors"
"log"
"os"
)

Go tiene una peculiaridad debido a que puede importar paquetes directamente desde github debido a que cada paquete tiene un nombre en especifico y este puede configurarse gracias a que go posee un fichero llamado 'go.mod' el cual permite que esto sea posible para este caso se importo el paquete de fiber y el CORS para configurar la posibilidad de ejecutar los metodos dentro del programa sin limitaciones de origen (Debido a que sin esto dificulta el hacer GET, POST, etc. Desde otras direcciones diferentes de localhost)

Variable Global#

FiberRoutes.go
var app = fiber.New()

La variable global app es utilizada a modo de configurar y llamar al servidor desde cualquier fichero dentro del paquete main

Configurando las rutas#

FiberRoutes.go
func FiberRoutes() {
// Default config
app.Use(cors.New(cors.Config{
Next: nil,
AllowOrigins: "*",
AllowMethods: "GET,POST,HEAD,PUT,DELETE,PATCH",
AllowHeaders: "",
AllowCredentials: false,
ExposeHeaders: "",
MaxAge: 0,
}))
app.Get("/id", FiberIdGET)
app.Get("/config/:id", FiberConfigGET)
app.Post("/config/:id", FiberConfigPOST)
app.Get("/result/:id", FiberResultGET)
app.Get("/log/:id", FiberLogGET)
port, ok := os.LookupEnv("PORT") // For Heroku or Elastic
if !ok {
port = "8080"
}
log.Fatal(app.Listen(":" + port))
}

Empezando por la ruta /id la cual debe de generar una ID para el cliente que funcionara como identificador unico para los siguientes procesos.
